Program Description:

This program maintains the itinerary header information.

/SCREEN1/

/SDESC1/

Program Operation:

The itinerary file maintenance program is a simple file maintenance program.

This information is added to the Land file, so the coding should be consistent with the land file coding:

First 3 letters: City code

Next letter : Type of land entry (I-Itinerary)

Next 2 letters : Sequence to keep codes unique

For example, the code: HKGI1 might be a paragraph describing arrival into Hong Kong.

The itinerary description can take up to two full lines. The city code should be the ARC airport code, or three letter abbreviation of the city if there is no airport.

The tour type must always be "I" for an Itinerary header.
